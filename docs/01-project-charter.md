# 01 项目启动说明

## 1. 项目名称
校园活动报名与签到管理系统

## 2. 项目目标
- 支持活动发布
- 支持学生报名
- 支持签到与统计
- 支持活动管理员发布、编辑、管理活动信息
- 支持学生在线查看活动并完成报名
- 支持报名信息审核与管理
- 支持现场二维码签到与记录
- 支持签到数据统计与结果导出
- 形成完整的项目管理文档与版本控制过程

## 3. 团队分工
请填写本组成员与角色。
#### 本项目为单人小组，由韩宝磊独立完成全部角色：
- 项目经理：负责整体计划、进度与协调
- 计划负责人：负责 WBS 分解、进度计划编制
- 配置管理员：负责 Git 仓库、分支策略、版本管理、冲突处理
- 开发 / 文档成员：负责所有文档编写、提交与合并
## 4. 项目边界
请写明纳入范围与不纳入范围。
#### 纳入范围：
- 活动信息发布与维护
- 学生在线报名功能
- 报名名单审核管理
- 签到记录与状态管理
- 签到统计与导出功能
- 项目管理全套文档（WBS、进度、配置、报告）
- Git 版本控制、分支管理、冲突解决
#### 不纳入范围：
- 支付、收费相关功能
- 与校园统一身份认证系统对接
- 高并发、服务器部署、上线方案
- 移动端 App 开发与设计
- 完整代码实现与数据库设计
## 5. 主要交付物
- GitHub 项目仓库与标准目录结构
- 01 项目启动说明
- 02 WBS 工作分解结构文档（三级）
- 03 简化进度计划文档
- 04 配置管理文档
- 05 实验总结报告
- 分支冲突模拟与解决记录
- 完整可追溯的 Git 提交历史
